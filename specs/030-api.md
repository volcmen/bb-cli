# 030 bb api

## Goal
Authenticated raw Bitbucket API passthrough (the `gh api` analog).

## Command surface
`bb api <PATH> [-X METHOD] [-f KEY=VALUE ...] [--paginate]`. Exit 0; non-zero on HTTP >= 400; 4 AuthError.

## Behavior & edge cases
- Auth via `crate::auth::header_for` (AuthError if none). Client `execute_raw(method, path, body)` (does NOT error on non-2xx — print the body, set exit code).
- `PATH`: `/2.0`-relative or absolute URL. `-f key=value` → JSON object body (string values); sent with `Content-Type: application/json`.
- Pretty-print the JSON response (fall back to raw text if not JSON).
- `--paginate`: GET only; follow body `next`, concatenate each page's `values` into one JSON array, print once.
- HTTP status >= 400 → print body to stderr/stdout and return SilentError (non-zero) — but still show the body.

## Tests
GET /user pretty JSON; -f builds JSON body (assert payload); 404 → body printed + non-zero; --paginate concatenates values; not-authed → AuthError.
